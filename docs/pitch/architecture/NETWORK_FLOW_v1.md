# NETWORK_FLOW_v1

## 1. Identidad del sistema
El flujo de red de Centinela representa una red operativa territorial diseñada para funcionar con conectividad limitada, sosteniendo persistencia operativa como prioridad.

## 2. Función operativa
Su función es transportar estado operativo desde nodos de campo hacia puntos de agregación y supervisión, preservando continuidad y legibilidad para decisión territorial.

## 3. Componentes / flujo
- Nodos de campo distribuidos.
- Gateway rural de agregación.
- Enlace de supervisión operativa.

Secuencia funcional:
1. El nodo registra condición local y estado operativo.
2. El estado se transmite según disponibilidad de enlace.
3. El gateway agrega y ordena señal territorial.
4. La supervisión recibe información utilizable para decisión.

## 4. Relación doctrinal
El flujo evita promesas de conectividad perfecta y asume restricciones reales de terreno. Refuerza soberanía tecnológica al modelar operación robusta con infraestructura mínima.

## 5. Relevancia para inversor
Hace visible la lógica de desriesgo operacional: no depende de cobertura ideal, sino de arquitectura resiliente y continuidad incremental. Esto permite evaluar factibilidad de pilotos y expansión por etapas.
