# VISUAL_STYLE_GUIDE_v1

## Objetivo
Definir criterios visuales mínimos para producir activos investor-ready coherentes con la doctrina territorial de Centinela del Monte.

## Estilo Visual General
- Enfoque: territorial, operativo y técnico.
- Tono visual: sobrio, funcional, creíble.
- Intención: mostrar infraestructura en contexto real, no producto de consumo.

## Estética Territorial/Rural/Técnica
- Priorizar escenas de campo con condiciones reales: polvo, intemperie, variación de luz, infraestructura austera.
- Mostrar escala territorial: distancia, cobertura, dispersión de nodos y puntos de enlace.
- Reforzar robustez operativa por encima de pulido comercial.

## Qué Evitar Visualmente
- Estética startup SaaS.
- Dashboards futuristas no realistas.
- Renders sci-fi o hiperconceptuales sin base operativa.
- Exceso de UI ocupando el foco narrativo.
- Estética “smart farming” genérica.

## Materiales y Entorno Sugeridos
- Materiales: metal pintado, polímeros resistentes, postes galvanizados, cajas estancas, cableado visible cuando corresponda.
- Entorno: caminos rurales, monte, bordes de parcela, instalaciones básicas, clima variable.
- Señalética: mínima, funcional y no publicitaria.

## Nivel de Realismo Esperado
- Medio-alto realismo técnico.
- Proporciones plausibles entre equipos y entorno.
- Iluminación natural o funcional de campo.
- Evitar estilización extrema o visual “concept art” dominante.

## Relación Visual entre Componentes
- Nodo: unidad de captura y persistencia en campo.
- Gateway: punto de agregación y enlace operativo.
- Territorio: condicionante principal de diseño y despliegue.
- Infraestructura: soporte físico para continuidad (postes, energía, montaje, mantenimiento).
- Regla: ningún componente se presenta aislado de su contexto operativo.

## Paleta Conceptual (Sin Branding Final)
- Base neutra-territorial: ocres, tierras, verdes apagados, grises técnicos.
- Acentos funcionales: azul profundo o cian moderado para señal/telemetría.
- Contraste: suficiente para legibilidad técnica, sin saturación comercial.

## Tipos de Diagramas Permitidos
- Arquitectura por capas.
- Flujo de red territorial.
- FSM simplificada para comunicación ejecutiva.
- Flujo de energía de alto nivel.
- Regla: diagramas explicativos, no decorativos.

## Estilo de Mapas
- Mapas conceptuales de cobertura y despliegue regional.
- Simbología simple: nodos, gateway, enlaces, zonas de prioridad.
- Escala y leyenda claras; evitar cartografía recargada.

## Estilo de Mockups
- Mockups operativos y sobrios, centrados en decisión de campo.
- UI mínima: alertas, estado, telemetría esencial.
- Evitar interfaces densas o estéticas de app de consumo.

## Reglas de Consistencia Visual
- Mantener misma lógica de escala y contexto entre assets.
- Usar terminología visual alineada con `PITCH_CANONICAL_LANGUAGE.md`.
- Priorizar continuidad narrativa entre renders, diagramas, mapas y mockups.
- Evitar contradicciones entre activos (ej. entorno, tipo de infraestructura, nivel técnico).
- Versionar activos con sufijo `_v1` y documentar uso previsto en deck.
