# VISUAL_ASSET_GUIDE_v1

## Objetivo
Definir la base oficial de activos visuales para producción investor-ready, asegurando coherencia con la narrativa territorial y la doctrina vigente del proyecto.

## Alcance
- Esta guía define **qué** visuales producir y **para qué**.
- No define estilo artístico final ni prompts de generación.
- No modifica arquitectura técnica ni firmware.

## Criterios de Consistencia Doctrinal
- Centinela se presenta como infraestructura territorial resiliente.
- Cada visual debe reforzar persistencia operativa y reducción de incertidumbre rural.
- Evitar framing de gadget de consumo o app aislada.
- Mantener trazabilidad entre narrativa inversor y base técnica existente.

## Lista Oficial de Visuales Requeridos

### 1) Nodo tipo caravana
- Propósito: representar la unidad de campo en contexto rural real.
- Mensaje: el nodo está pensado para operación persistente en territorio hostil.
- Relación doctrinal: infraestructura territorial resiliente y continuidad operativa.

### 2) Nodo abierto con componentes
- Propósito: mostrar legibilidad técnica del dispositivo y sus bloques funcionales.
- Mensaje: diseño orientado a robustez, mantenibilidad y trazabilidad técnica.
- Relación doctrinal: soberanía tecnológica y arquitectura verificable.

### 3) Gateway rural
- Propósito: visualizar el punto de agregación/comunicación en despliegue territorial.
- Mensaje: la red no depende de infraestructura urbana densa para operar.
- Relación doctrinal: persistencia operativa en entornos de baja conectividad.

### 4) Poste/antena LoRa
- Propósito: representar el elemento físico de enlace en campo.
- Mensaje: conectividad de largo alcance adaptada a geografía rural.
- Relación doctrinal: reducción de incertidumbre rural mediante enlaces resilientes.

### 5) Flujo de red territorial
- Propósito: explicar circulación de señales desde nodos a capa de supervisión.
- Mensaje: arquitectura distribuida con lógica operacional clara.
- Relación doctrinal: sistema de persistencia operativa y gobernanza técnica.

### 6) Mapa conceptual de cobertura
- Propósito: visualizar alcance operativo esperado a nivel territorial (conceptual).
- Mensaje: planificación de despliegue por zonas y prioridades de operación.
- Relación doctrinal: enfoque territorial, no centrado en dispositivo aislado.

### 7) FSM simplificada
- Propósito: comunicar comportamiento operativo del nodo de forma comprensible.
- Mensaje: operación determinística con estados definidos y manejo de criticidad.
- Relación doctrinal: disciplina de ingeniería, robustez y continuidad del sistema.

### 8) Arquitectura general del sistema
- Propósito: sintetizar capas principales (nodo, enlace, operación, gobernanza).
- Mensaje: solución sistémica, no componente único.
- Relación doctrinal: separación entre core técnico y narrativa investor-ready.

## Orden Sugerido de Producción
1. Arquitectura general del sistema
2. Flujo de red territorial
3. Mapa conceptual de cobertura
4. Nodo tipo caravana
5. Nodo abierto con componentes
6. Gateway rural
7. Poste/antena LoRa
8. FSM simplificada
